# Weapons Classification Flowchart Structure

## Overview

The weapons classification system uses a hierarchical tree structure to organize taxonomical definitions. The workflow separates **structure** from **metadata** to keep things organized and maintainable.

## File Organization

### 1. **flowchart-structure.txt** (User Edits This)
Plain text file using box-drawing characters that defines the **taxonomy hierarchy**.

**Format:**
```
NodeId
├── ChildNode (decision: Option A vs Option B)
│   ├── LeafNode (classification)
│   └── AnotherLeaf (classification)
└── AnotherChild (decision: Choice 1 vs Choice 2)
    ├── FinalClassification (classification)
    └── AnotherClassification (classification)
```

**Rules:**
- Node IDs must be alphanumeric + underscore only (e.g., `SemiAutomaticRifles`, not `Semi Automatic Rifles`)
- Space allowed between node ID and parentheses: `NodeId (type)`
- Types: `decision` or `classification`
- Decision nodes have metadata: `(decision: description of choice)`
- Classification nodes are terminal: `(classification)` or no annotation
- Use proper box-drawing characters for hierarchy: `├──`, `└──`, `│`
- Root node must have no indentation

**Example:**
```
SmallArms
├── Handguns_BarrelType (decision: Rifled vs Smooth Bore)
│   ├── Handguns_Rifled (decision: Self-Loading Pistol vs Revolver)
│   │   ├── Rifled_SelfLoadingPistols (classification)
│   │   └── Rifled_Revolvers (classification)
│   └── SmoothBore_OtherHandguns (classification)
└── LongGuns (decision: Rifled vs Smooth Bore)
    ├── LongGuns_Rifled (decision: SMG vs MPMG vs Rifles)
    │   ├── Rifled_SubMachineGuns (classification)
    │   └── Rifles (decision: Self-Loading vs Manually-Loaded)
    └── LongGuns_SmoothBore (classification)
```

### 2. **flowchart-metadata.json** (User Edits This)
JSON file containing all **labels, descriptions, and images**.

**Structure:**
```json
{
  "nodes": {
    "NodeId": {
      "label": "Display label with spaces and formatting",
      "img_src": "https://url/to/image.png"
    }
  },
  "edge_labels": {
    "ParentNode->ChildNode": {
      "label": "Edge label",
      "description": "Full description of this choice",
      "img_src": "https://url/to/edge/image.png"
    }
  }
}
```

**Notes:**
- Node IDs match those in `flowchart-structure.txt`
- Edge keys use format: `SourceNode->TargetNode`
- Labels can have spaces and special characters
- Images are optional (use empty string if not available)
- Descriptions only appear on edges, not on nodes

### 3. **flowchart-structure.json** (Generated)
Intermediate JSON representation of the tree structure. **Do not edit manually** - generated by `parse_structure.py`.

## Workflow

### Step 1: Edit Structure
Update `flowchart-structure.txt` to change the taxonomy hierarchy:

```bash
# Edit the file
vim flowchart-structure.txt
```

### Step 2: Validate Structure
Validate the structure file for syntax errors and consistency:

```bash
python3 src/validate_structure.py
```

**Checks:**
- ✅ Valid node IDs (alphanumeric + underscore)
- ✅ No duplicate node IDs
- ✅ Proper indentation hierarchy
- ✅ Decision nodes have explanatory metadata
- ✅ No indent jumps > 1 level

**Example output:**
```
✅ Validation passed!
   Total nodes: 28
   Decision nodes: 7
   Classification nodes: 21
```

### Step 3: Parse Structure
Convert the text structure to JSON:

```bash
python3 src/parse_structure.py
```

This generates `flowchart-structure.json` from `flowchart-structure.txt`.

### Step 4: Edit Metadata
Update `flowchart-metadata.json` with labels, descriptions, and images:

```bash
# Edit the file
vim flowchart-metadata.json
```

### Step 5: Generate Mermaid Diagram
Create the final Mermaid flowchart:

```bash
python3 src/generate_mermaid.py
```

This generates `weapons-classification-flowchart.mmd` with enriched metadata.

## Complete Workflow Commands

```bash
# 1. Validate structure
python3 src/validate_structure.py

```

## Valid Node Types

### Decision Nodes
- Intermediate nodes that ask a question
- Have 2 or more children
- Metadata explains the decision: `(decision: Option A vs Option B)`
- Format in structure file:
  ```
  Handguns_BarrelType (decision: Rifled vs Smooth Bore)
  ```

### Classification Nodes
- Terminal nodes representing final weapon classifications
- Have no children
- Optional metadata: `(classification)` or no annotation
- Format in structure file:
  ```
  Rifled_SelfLoadingPistols (classification)
  ```

### 1. Edit `flowchart-structure.txt`
```
├── Rifles (decision: Self-Loading vs Manually-Loaded)
│   ├── SelfLoadingRifles (decision: Auto vs Semi)
│   │   ├── AutomaticRifles (classification)
│   │   └── SemiAutomaticRifles (classification)  ← Add new node here
│   │   └── NewClassification (classification)    ← Like this
```

### 2. Edit `flowchart-metadata.json`
```json
{
  "nodes": {
    "NewClassification": {
      "label": "Display Name for New Classification",
      "img_src": "https://url/to/image.png"
    }
  },
  "edge_labels": {
    "SelfLoadingRifles->NewClassification": {
      "label": "New Option",
      "description": "Description of this classification choice",
      "img_src": "https://url/to/edge/image.png"
    }
  }
}
```

### 3. Validate and Generate
```bash
python3 src/validate_structure.py
python3 src/parse_structure.py
python3 src/generate_mermaid.py
```

## Troubleshooting

### "Invalid node ID" error
- Node IDs must contain only letters, numbers, and underscores
- ❌ `Semi Automatic Rifles` → ✅ `SemiAutomaticRifles`
- ❌ `Rifle-Action` → ✅ `RifleAction`

### "Duplicate node ID" error
- Each node ID must be unique across the entire tree
- Check if you've defined the same node twice

### "Indent jump too large" error
- Indentation must increase by exactly 1 level
- ❌ Jumping from level 0 to level 2
- ✅ Go level 0 → 1 → 2 → 3

### Missing metadata error
- Ensure all node IDs in `flowchart-structure.txt` have entries in `flowchart-metadata.json`
- New nodes won't generate properly without metadata

## Tips

1. **Use consistent naming**: `CamelCase` for node IDs, descriptive names in labels
2. **Keep decisions clear**: Metadata should clearly explain the binary/multiple choice
3. **Use meaningful images**: Link to visual references for each classification
4. **Test after changes**: Always run `validate_structure.py` before generating
5. **Version control**: Commit structure changes separately from metadata updates


```bash
python3 src/generate_mermaid.py
```

1. Add node to `flowchart-structure.txt` under parent decision node
2. Add entry to `flowchart-metadata.json` with label and image
3. Run validation and generation:
   ```bash
   python3 src/validate_structure.py && \
   python3 src/parse_structure.py && \
   python3 src/generate_mermaid.py
   ```

## I want to verify my changes are valid

```bash
python3 src/validate_structure.py
```

This checks:
- ✅ Node IDs are valid
- ✅ No duplicate IDs
- ✅ Tree hierarchy is correct
- ✅ All decisions have explanatory metadata

| "Duplicate node ID" | Same ID used twice | Make IDs unique |
| "Indent jump too large" | Skipped indentation levels | Add intermediate nodes |
